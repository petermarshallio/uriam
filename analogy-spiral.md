# Uriam — The Spiral Analogy

*One optional way to visualize Uriam's cycle — not the framework's name, and not required reading to use it. For the framework itself, see [`fundamentals.md`](./fundamentals.md); for strict definitions, see [`reference.md`](./reference.md).*

## Not a spiral that grows

A generic spiral — a nautilus shell, for instance — grows bigger with every turn. That's a beautiful image, but it isn't quite honest about what this cycle does. The shape here is closer to a staircase: the same radius, the same four moves, on every turn.

That means you're not "spiraling" in the usual sense — spinning out, losing control. A spiral *staircase* is purposeful. It requires effort. It's about elevation, one disciplined step at a time.

## Spirals all the way down

Every stage of the cycle is itself a full copy of the same cycle — the Fractal Property, described properly in `fundamentals.md`. Pictured with the staircase image in mind, that property has a name worth keeping around, even once "Spiral" stopped being the framework's own name:

> "It's spirals all the way down."

The Internalising stage of a project *is* its own small climb: you interact with small experiments to learn from, internalise what they teach you, examine what you found, and externalise the next experiment. The Examining stage of a decision is the same small climb, one level in: you interact with hypotheses to see what they imply, internalise what that reveals, and externalise which one to test next.

This isn't a metaphor stacked on a metaphor — it's structurally true regardless of which image you use to picture it. The staircase just happens to be a vivid way to hold onto "the same shape, however far you zoom."

## Where this analogy holds, and where it doesn't

Worth being honest about the limits of the image, same as any analogy:

- **Good for:** momentum (climbing = progress), the fractal property (a staircase inside a staircase), and the discipline of a fixed shape repeating — the same four moves, not a different four each time.
- **Weak for:** anything relational — two people, or a person and an AI, moving through their own cycles at once. A spiral doesn't have a natural vocabulary for how two independent climbs relate to each other. (See `analogy-production.md`'s musical framing — unison, dissonance, counterpoint, suspension — for that instead.)
- **Not required at all.** Uriam's actual fundamentals live in [`bin/profile/primitives.md`](./bin/profile/primitives.md) and reference no image whatsoever — Actor, Role, Node, Edge, Uriam Graph. The spiral is one way of picturing the cycle for a reader who finds a physical shape easier to hold onto than pure structure. Use it if it helps. Drop it if it doesn't.
